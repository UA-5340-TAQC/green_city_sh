**Test Case ID:** TC-02  
**Title:** Verify presence of mandatory UI elements and data formats in the first event card  
**Related Requirement/User Story:** User story #Events-List-UI  
**Author:** Voloshyn Volodymyr  
**Priority:** Medium

**Preconditions:**

1. User is authorized on the GreenCity website.
2. Events page is loaded: `https://www.greencity.cx.ua/#/greenCity/events`.
3. At least one event card is present in the list (`app-events-list-item`).

**Test Steps:**
| Step # | Step Description | Test Data | Expected Result |
|:------:|:----------------|:----------|:---------------|
| 1 | Locate the first event card in the list | `.app-events-list-item` | The event card is visible and correctly rendered |
| 2 | Verify the event image visibility | `img` element | Image is present, loaded, and has a valid source |
| 3 | Validate the event category tag | `ul.ul-eco-buttons` | Tag text matches one of: "Environmental", "Social", or "Economic" |
| 4 | Check the date and time display format | `.date-container` | Date and time are visible (e.g., "Apr 9, 2026 \| 4:00 PM") |
| 5 | Verify the event status label | `.event-status` | Status text is strictly "Open" or "Closed" |
| 6 | Verify the presence of action buttons | | "More" and "Edit event" buttons are displayed and enabled |

**Postconditions:**

1. Event list remains unchanged.

**Status:** Not Run

**Environment:**

- **OS:** Windows 10
- **Browser:** Chromium, Firefox, WebKit
- **Version:** Latest

**Screenshots:** No screenshots yet (test not executed).

**Additional Context:**

- **Card selector:** `app-events-list-item:first-child`
- **Image selector:** `img[_ngcontent-ng-c946938428]`
- **Category tag:** `span.tag-active` inside `ul-eco-buttons`
- **Status selector:** `div.event-status` (with `lock-open` or `lock` icons)
- **Buttons class:** `.secondary-global-button.m-btn`
