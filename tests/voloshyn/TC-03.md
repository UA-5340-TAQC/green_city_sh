**Test Case ID:** TC-03  
**Title:** Verify that Date Range filter correctly filters events  
**Related Requirement/User Story:** User story #Events-Filtering  
**Author:** Voloshyn Volodymyr  
**Priority:** High

**Preconditions:**

1. User is authorized on the GreenCity website.
2. User is on the Events page: `https://www.greencity.cx.ua/#/greenCity/events`.
3. Multiple events exist with various dates scheduled for the current month.

**Test Steps:**
| Step # | Step Description | Test Data | Expected Result |
|:------:|:----------------|:----------|:---------------|
| 1 | Click on the "Date range" filter dropdown | `mat-date-range-input` | A calendar window (datepicker) is displayed |
| 2 | Select the start date of the range | (e.g., April 1, 2026) | Start date is accepted and highlighted in the calendar |
| 3 | Select the end date of the range | (e.g., April 30, 2026) | End date is accepted; the date range is applied to the view |
| 4 | Verify the filtered results and "Items found" label | | Only events within the April 2026 range are shown; count is updated |

**Postconditions:**

1. The Date Range filter remains active.
2. Results are dynamically updated based on the selected range.

**Status:** Not Run

**Environment:**

- **OS:** Windows 10
- **Browser:** Chromium, Firefox, WebKit
- **Version:** Latest

**Screenshots:** No screenshots yet (test not executed).

- **Current month:** April 2026
- **Main selector:** `mat-date-range-input`
- **Calendar toggle:** `mat-datepicker-toggle`
- **Results counter:** `.items-found` (Verify the number matches the visible cards)
